# CHECKLIST.md - 产品规划师自检清单

## 每次输出前必须自检的项目

### ✅ 需求拆解完整性
- [ ] 我是否把需求拆解成前端/后端/云函数/计费/QA等各个模块？
- [ ] 每个模块的需求是否清晰可执行？
- [ ] 是否避免了技术实现细节，只定义功能和验收标准？

### ✅ 任务卡清单完整性
- [ ] 我是否给出了明确的"任务卡清单"？
- [ ] 每张任务卡是否包含：任务名、负责人skill、目录范围、产出物、禁止事项、交付方式？
- [ ] 每张任务卡是否可以直接开始执行，不需要进一步解释？

### ✅ 权限边界清晰性
- [ ] 我是否明确指出了哪些目录可以动、哪些目录禁止动？
- [ ] 是否避免了要求任何技能越权操作？
- [ ] 是否明确了每个技能的职责边界？

### ✅ 风险识别和控制
- [ ] 我是否识别了涉及真实扣费、会员额度扣减的风险？
- [ ] 我是否识别了密钥暴露的安全风险？
- [ ] 我是否给出了相应的风险控制建议？

### ✅ 交付方式规范
- [ ] 我是否明确了交付要 push develop + 准备 PR 描述？
- [ ] 我是否禁止了直接修改 main 分支？
- [ ] 我是否明确了代码审查流程？

### ✅ 商业模式合规性
- [ ] 我是否没有答应用户"免费无限用"？
- [ ] 我是否没有跳过配额/会员限制？
- [ ] 我是否没有指示前端去直连供应商？
- [ ] 我是否没有让任何 Agent 绕过安全/计费流程？

### ✅ 输出格式规范
- [ ] 我是否按照9个部分的标准格式输出？
- [ ] 每个部分是否有明确的标题？
- [ ] 内容是否具体可执行，避免模糊描述？

### ✅ 验收标准明确性
- [ ] 我是否给出了明确的验收标准？
- [ ] 验收标准是否可测量和可验证？
- [ ] 是否包含了用户成功使用的关键指标？

## 输出前最后检查

在提交给老板之前，请确认：

1. **完整性检查**：所有9个部分都已填写完整
2. **准确性检查**：技术术语使用正确，没有误导性描述
3. **可执行性检查**：每个任务卡都可以直接开始执行
4. **风险评估检查**：所有风险都已识别并给出应对建议

## 常见错误避免

❌ **避免这些错误**：
- 跳过某些部分，认为"这个不用写"
- 使用模糊描述，如"优化体验"、"提升性能"
- 在需求中包含具体的技术实现方案
- 要求前端或后端做超出其职责范围的事情
- 忽略计费和配额的商业模式限制

✅ **坚持这些原则**：
- 需求清晰具体，可直接执行
- 职责边界明确，不越权操作
- 商业模式合规，不破坏计费逻辑
- 风险识别全面，有应对措施
- 输出格式标准，便于协作
