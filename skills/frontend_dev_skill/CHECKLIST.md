# CHECKLIST.md - 前端开发工程师自检清单

## 每次代码提交前必须自检的项目

### ✅ 需求实现完整性
- [ ] 我是否完整实现了产品需求中的所有页面功能？
- [ ] 所有用户交互流程是否都能正常工作？
- [ ] 是否按照设计稿实现了UI布局和样式？
- [ ] 响应式设计是否在所有设备尺寸上正常显示？

### ✅ 组件设计和代码质量
- [ ] 组件是否按照原子设计原则进行了合理拆分？
- [ ] 组件是否具有良好的复用性和可维护性？
- [ ] 是否使用了TypeScript类型定义，没有any类型？
- [ ] 组件的props是否有明确的类型定义和默认值？

### ✅ 状态管理和数据流
- [ ] 全局状态是否正确使用了Zustand进行管理？
- [ ] API调用是否使用了React Query进行数据获取和缓存？
- [ ] Loading状态和Error状态是否都有合适的UI处理？
- [ ] 表单数据是否使用React Hook Form进行了合理管理？

### ✅ 用户交互和体验
- [ ] 所有按钮点击是否有正确的响应和反馈？
- [ ] 异步操作期间是否显示了Loading状态？
- [ ] 错误信息是否使用Ant Design的组件友好显示？
- [ ] 页面跳转是否使用Next.js的App Router？

### ✅ 文件上传和媒体处理
- [ ] COS上传是否使用了STS临时密钥，没有暴露密钥？
- [ ] 文件上传是否显示进度条和状态反馈？
- [ ] 图片上传后是否能立即预览效果？
- [ ] 文件类型和大小是否在前端进行了验证？

### ✅ 会员权限和安全
- [ ] 是否检查了用户的登录状态？
- [ ] 会员状态和配额余额是否正确显示？
- [ ] 权限不足时是否有合适的提示和引导？
- [ ] 敏感信息（如API密钥）是否没有暴露在前端代码中？

### ✅ 性能和优化
- [ ] 是否进行了代码分割，避免打包文件过大？
- [ ] 图片是否进行了适当的压缩和格式优化？
- [ ] 是否使用了React.memo或useMemo进行性能优化？
- [ ] 页面加载时间是否在可接受范围内？

### ✅ 测试和兼容性
- [ ] 是否在Chrome、Firefox、Safari等主流浏览器上测试过？
- [ ] 移动端和桌面端的显示效果是否都正常？
- [ ] 用户操作的主要流程是否都进行了测试？
- [ ] 边界情况和异常场景是否都有处理？

### ✅ 代码规范和提交
- [ ] 代码是否通过了ESLint检查，没有警告或错误？
- [ ] 代码是否使用Prettier进行了格式化？
- [ ] Git提交信息是否遵循了项目的提交规范？
- [ ] 是否只修改了任务范围内的文件？

## 输出前最后检查

在提交代码之前，请确认：

1. **功能完整性**：所有产品需求都已实现并测试通过
2. **用户体验**：所有交互都有反馈，错误处理友好
3. **代码质量**：TypeScript类型安全，组件设计合理
4. **性能表现**：页面加载速度快，交互响应及时
5. **安全性**：没有暴露敏感信息，权限控制正确

## 常见错误避免

❌ **避免这些错误**：
- 在前端代码中硬编码API密钥或敏感信息
- 忽略Loading状态，让用户不知道操作是否在进行
- 没有处理Error状态，出现错误时页面白屏
- 直接使用any类型，没有TypeScript类型保护
- 没有进行响应式设计，移动端体验差
- 组件职责不清，一个组件做得太复杂

✅ **坚持这些原则**：
- 用户体验优先，所有操作都要有反馈
- 代码质量优先，使用TypeScript保证类型安全
- 性能优先，使用适当的优化手段
- 安全优先，不暴露敏感信息
- 可维护性优先，组件设计清晰合理
